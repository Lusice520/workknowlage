# UI and UX Rules

- WorkKnowlage 应该像 focused local productivity tool，不像 marketing site。
- 优先 dense but readable 的 operational interfaces。
- Editor 和 shell interactions 要 predictable。
- 避免和 document content 抢注意力的 decorative UI。
- Search、navigation、sidebars、editor controls 优先考虑 scanning、keyboard flow、state clarity。
- 改变 layout、focus 或 interaction behavior 时，用 browser / app-level checks 验证 visible changes。
