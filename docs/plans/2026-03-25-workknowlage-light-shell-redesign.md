# WorkKnowlage Light Shell Redesign

**Date:** 2026-03-25

## Goal

将当前 `WorkKnowlage` 第一阶段三栏壳子从“偏厚重展示稿”调整为“轻编辑桌面”，并满足以下明确要求：

- 整体更轻，不笨重
- 整页一屏展示
- 不出现任何滚动条
- 保持三栏工作台结构
- 保留未来接入 `WorkPlan` 编辑器和 SQLite 数据层的接入口

## Problem Summary

当前版本的笨重感主要来自以下几个方面：

- 三栏都使用了较大的圆角、卡片层和阴影层
- 标题、正文、侧栏信息的字号整体偏大
- 左栏和中栏存在纵向滚动容器
- 左栏月历卡和右栏分享卡占用了过多垂直空间
- 中栏示例内容更像视觉展示稿，而不是一屏内的工作台摘要态

## Redesign Direction

本次采用“编辑化轻桌面”方向：

- 保留桌面知识库工作台的真实结构
- 通过更克制的排版、更轻的边框和更紧的密度降低厚重感
- 将内容控制为“一屏摘要态”，而不是试图模拟整篇长文

这次不是简单缩放，而是整体切换到更接近 `Notion / Craft / Agenda` 一类的轻桌面编辑气质。

## Layout Strategy

### Overall Shell

- 页面高度锁定在当前窗口高度内
- 不允许页面级滚动
- 三栏布局改为更紧凑的宽度与更小的外边距

推荐比例：

- 左栏：约 `272px`
- 中栏：自适应
- 右栏：约 `296px`
- 外层间距：约 `14px`

### Left Sidebar

- 保留空间卡、新建入口、快捷入口、文档树、底部工具
- 删除大体积月历卡
- 用更轻的迷你信息块替代底部日历展示
- 树节点采用更薄、更扁、更轻的激活样式

### Center Pane

- 主标题明显缩小一档
- 元信息改为更轻的辅助层
- 正文只展示一屏摘要内容
- 内容结构保留为：
  - 标题
  - 一行元信息
  - 两段正文
  - 一个引用块
  - 一组紧凑媒体卡
  - 一个低存在感 `EditorHost`

### Right Sidebar

- 保留大纲、标签、反向链接、分享/导出入口
- 各模块改成紧凑摘要态
- “分享与导出”从大卡片改为轻量工具区

## Typography Strategy

不额外引入网络字体，继续使用系统中文字体栈，以稳定和桌面原生感为优先。

重点调整：

- 标题：从展示级超大字改成编辑页标题级
- 正文：从偏视觉稿尺寸改成真实阅读尺寸
- 左栏和右栏：统一降低字号与字重
- 辅助信息：明确退到信息层，不与正文争抢视觉焦点

目标不是更花哨，而是：

- 更轻
- 更克制
- 更易读
- 更像真实应用而不是概念稿

## Visual System Adjustments

- 减小圆角层级
- 阴影只保留极浅一层
- 去掉明显玻璃感和过重的 blur 感
- 边框更轻，但保留结构感
- 收窄模块间垂直间距
- 减少卡片套卡片

## No-Scroll Strategy

为了严格满足“一屏无滚动条”，采取以下策略：

- 页面级 `overflow` 关闭
- 左栏、中栏、右栏都不使用 `overflow-y-auto`
- 用固定高度预算设计每个区域
- 中栏示例内容改成摘要态，而不是长文
- 左栏底部区域改为轻量模块，避免把树区压缩出滚动

## Scope Of Change

本次只调整视觉和布局，不改：

- 数据模型
- preload API 形状
- Electron 运行方式
- `EditorHost` 的未来替换边界

## Acceptance

1. 运行窗口时整页一屏，不出现滚动条
2. 整体比当前版本明显更轻、更平、更紧凑
3. 三栏仍有真实信息结构与状态联动
4. 版面焦点回到中栏标题和正文
5. 后续接入真实数据和复用编辑器时无需重做当前布局
