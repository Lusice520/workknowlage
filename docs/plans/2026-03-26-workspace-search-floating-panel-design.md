# Workspace Search Floating Panel Design

**Goal**

让左侧搜索结果以浮动面板形式显示，不参与左侧栏高度计算。

**Current Problem**

- 当前搜索结果直接在文档流里展开。
- 一旦出现加载态、空态或结果列表，就会把左侧栏内容整体向下挤，影响日历和底部区域。
- 这和搜索入口“工具型控件”的角色不匹配。

**Decision**

- 搜索框维持当前位置不变。
- 结果面板改为锚定在搜索框下方的绝对定位浮层。
- 面板宽度跟随搜索框，使用固定最大高度和内部滚动。

**Interaction**

- 默认态：只显示单行搜索框。
- 输入后：在输入框下方弹出搜索结果面板。
- 面板内部滚动，不改变左侧栏高度。
- 键盘上下选择、回车打开、Esc 取消高亮行为保持不变。

**Why This Approach**

- 直接解决“展开后不应该影响左侧栏高度”的问题。
- 不需要引入 portal，定位与维护都更简单。
- 保持现有搜索交互，只调整展示层结构，风险低。
