# Collection Views Design

**Goal**

完成第一期“知识入口成型”产品迭代，把左栏 `所有笔记 / 收藏夹` 从静态占位升级为可工作的集合页入口，同时补齐收藏字段与工作区快照加载能力。

**Current Problem**

- 左栏快捷入口来自默认假数据，目前只有视觉存在感，没有真正交互和状态语义。
- 当前工作区加载使用“先列文件夹，再按文件夹逐个列文档”的 N+1 方式，后续文档变多会明显拖慢切空间和刷新。
- 用户只能从文件树进入文档，不适合按“最近编辑 / 收藏文档”这类非树状工作流浏览内容。

**Decision**

采用“左栏入口 + 中栏集合页”的方案，而不是把快捷入口做成左栏树筛选。

原因：

- 不打断现有文件树心智。文件树继续表达结构归属，集合页表达工作视图。
- 更容易继续扩展到 `最近编辑`、`按标签浏览`、`最近分享` 这类后续视图。
- 中栏本来就是工作主视区，让集合页承接列表态更自然。

## Product Shape

### Left Sidebar

- 保留当前 `所有笔记 / 收藏夹` 两个入口位置。
- 点击后不切换文件树结构，而是进入对应集合页。
- 当前激活入口要有明确高亮，且与文档树选中态互斥。

### Center Pane

新增两种集合页：

- `所有笔记`
- `收藏夹`

集合页展示：

- 标题
- 当前空间名
- 文档总数
- 文档列表

每条文档项展示：

- 标题
- 摘要预览
- 所在文件夹
- 最近更新时间
- 标签
- 收藏按钮

点击文档项后进入真实文档编辑页。

### Favorites

收藏是文档级真实状态，不是临时前端 UI 状态。

- 文档支持 `isFavorite`
- 在集合页列表项可直接收藏/取消收藏
- 后续也应允许在文档页头部复用这一动作，但这一期不强制要求把按钮铺到所有入口

## Data Model

### Document

在 `documents` 表新增：

- `is_favorite INTEGER NOT NULL DEFAULT 0`

前端 `DocumentRecord` 同步新增：

- `isFavorite: boolean`

### Workspace State

在 `WorkspaceState` 新增集合页状态：

- `activeCollectionView: 'tree' | 'all-notes' | 'favorites'`

约束：

- 打开文档时自动回到 `tree`
- 点击快捷入口时清空激活文档的中心视图，但不破坏当前文件树展开状态
- 切换空间时默认回到 `tree`

## Data Loading

新增 `workspace snapshot` API，一次返回当前 space 的：

- folders
- documents

目标：

- 替代当前 `folders.list + documents.list(root) + documents.list(each folder)` 的 N+1 请求
- 成为后续集合页、树视图、统计信息的统一数据入口

这期不把 quick notes 并入 snapshot，避免把“日期驱动快记”和“正式文档集合”混成一个模型。

## Sorting Rules

集合页默认按最近更新时间倒序。

一期内不做：

- 手动排序
- 多维筛选
- 标签筛选

## Empty States

- 所有笔记为空：提示创建第一篇文档
- 收藏夹为空：提示先去收藏一些文档

## Out Of Scope

- 把每日快记混入所有笔记
- 最近编辑视图
- 标签筛选视图
- 收藏夹文件夹
- 拖拽排序
- 多选批量操作

## Acceptance

1. 左栏 `所有笔记 / 收藏夹` 可点击并进入中栏集合页。
2. 集合页能展示当前空间文档列表，并能打开具体文档。
3. 收藏状态真实持久化，重新加载应用后仍保留。
4. 切换空间时集合页与文档数据一起正确刷新。
5. 工作区加载改为 snapshot API，不再使用按文件夹逐个拉文档的 N+1 方式。
