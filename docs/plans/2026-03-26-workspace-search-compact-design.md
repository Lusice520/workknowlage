# Workspace Search Compact Design

**Goal**

把左侧搜索区收成单行搜索入口，减少默认态纵向占用，并把输入字体稳定压到 12px。

**Current Problem**

- 默认态搜索区有外层卡片、较大内边距和一行辅助文案，整体显得过高。
- 输入框虽然写了 `text-[12px]`，但实际视觉仍偏大，用户感知接近 16px。
- 搜索入口在侧栏顶部，应该更像工具控件，而不是一张内容卡片。

**Decision**

- 默认态仅保留一行搜索框。
- 移除“输入关键词，快速查找文档或快记。”辅助文案。
- 弱化外层卡片感，只保留紧凑输入框作为主要视觉单位。
- 用更明确的输入样式约束把字体固定为 12px。

**Interaction**

- 未输入时：只显示单行搜索框。
- 输入后：在搜索框下方展开加载态、空态或结果列表。
- 键盘上下选择、回车打开、Esc 取消高亮行为保持不变。

**Why This Approach**

- 直接回应“太占空间”的反馈，默认态高度立刻下降。
- 把搜索入口还原为工具属性，符合左侧栏信息密度。
- 不改搜索逻辑，只改默认态展示与输入样式，风险低。
