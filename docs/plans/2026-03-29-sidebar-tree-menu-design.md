# Sidebar Tree Menu Design

**Date:** 2026-03-29

## Goal

调整左侧目录树交互，让“新建”与“重命名/删除”分组展示，并补上文档节点上的新增入口，为后续继续扩展菜单项留出结构。

## Current Context

- 根目录当前直接暴露两个按钮：`新建文件`、`新建文件夹`。
- 文件夹节点当前直接暴露四个动作：`重命名`、`新建文件`、`新建文件夹`、`删除`。
- 文档节点当前只暴露两个动作：`重命名`、`删除`。
- 数据模型里只有 `folder.parentId` 和 `document.folderId`；文档不是容器节点，不能直接挂子节点。

## Options

### Option A: 菜单化现有动作，文档节点新增“同级新增”入口

- 根目录 `+` 改为一个创建菜单，先放 `新建文件`、`新建文件夹`。
- 文件夹节点保留两个 icon：
  - `+` 打开创建菜单
  - `更多` 打开 `重命名`、`删除`
- 文档节点也增加同样的两个 icon：
  - `+` 在当前文档所在层级创建新文档/文件夹
  - `更多` 打开 `重命名`、`删除`

优点：不改后端模型，改动小，菜单结构也方便后续继续加动作。

缺点：文档节点上的“新增”是同级新增，不是真正的“文档可嵌套子节点”。

### Option B: 让文档成为可容器节点

- 给文档引入父子关系字段。
- 重写目录树选择器、拖拽、创建、删除和持久化结构。

优点：完全贴合“文档下面挂子节点”的语义。

缺点：影响前后端 schema、迁移、排序和拖拽，明显超出本次最小需求。

## Recommendation

采用 **Option A**。

这是当前代码结构下最小且稳定的方案，也和你希望“后续还会增加别的”一致：我们先把交互入口抽成菜单，再保留数据模型边界不动。

## Interaction Details

- 根目录：
  - 显示一个 `+` 菜单按钮。
  - 菜单项先放 `新建文件`、`新建文件夹`。
- 文件夹节点：
  - `+` 菜单：`新建文件`、`新建文件夹`
  - `更多` 菜单：`重命名`、`删除`
- 文档节点：
  - `+` 菜单：在 `sourceFolderId` 层级执行 `新建文件`、`新建文件夹`
  - `更多` 菜单：`重命名`、`删除`

## Testing Scope

- 根目录 `+` 菜单可打开并创建根级文档。
- 文件夹和文档的 `更多` 菜单能触发重命名、删除。
- 文档节点的 `+` 菜单能对当前层级创建文档/文件夹。
