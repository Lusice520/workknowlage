# Graphify Sidebar Association Design

**Goal**

在不重构 WorkKnowlage 主布局的前提下，把 Graphify 型“关系发现能力”接入现有文档体验。

这次设计的重点不是先做一个完整图谱页，而是：

- 保留现有右侧栏的目录价值
- 保留显式关系的上下文信息
- 新增一张独立的 `关联` 卡片，承接系统发现的新关系
- 为目录中的彩色状态点补上明确语义

## Current Product Baseline

当前右侧栏已经不是空白状态，而是一个有明确结构的真实模块：

- 顶部有标签区
- 中部有 `文稿概览`，承担目录导航
- 底部有 `关联引用`，展示 mention / backlink 显式关系

现有实现已经说明两件事：

1. 用户已经在右侧栏形成了“看结构 + 看关系”的使用习惯。
2. 新能力更适合以增量方式接入，而不是先重做主导航或三大主视图。

因此这次设计选择“最小侵入式方案”。

## Problem Statement

如果直接把 Graphify 理念完整搬进现有产品，最容易出现两个问题：

- 右侧栏从“辅助阅读区”膨胀成“第二主界面”
- 目录、显式关系、智能关系混在一起，用户无法快速判断“这是已有关系”还是“系统推荐关系”

同时，当前目录里的彩色点已经出现了一个很有价值的方向：目录不再只是跳转列表，而是带状态的“增强目录”。但如果不给颜色固定语义，这些点会更像装饰而不是信息编码。

## Approaches Considered

### Option A. 重做主界面，做 `文档 / 搜索 / 图谱` 一级导航

优点：

- 概念完整
- 容易承载完整图谱产品叙事

问题：

- 改动范围过大
- 会打断当前文档流
- 不适合这一期直接落地

### Option B. 保持两个右侧模块，把 Graphify 信息并入一个总的“知识上下文”

优点：

- UI 改动最少
- 可以较快塞进现有结构

问题：

- 显式关系和智能关联会混淆
- 后续越做越重时，卡片语义会越来越不清楚

### Option C. 保持现有目录和显式关系，再新增一张独立 `关联` 卡片

优点：

- 保留现有使用路径
- 新旧关系职责清晰
- 后续可以自然演进到搜索关联和局部图

问题：

- 需要定义卡片边界和状态切换

**Recommendation**

选择 Option C。

这是当前最稳的切片：不改主导航、不推翻右侧栏，只把 Graphify 型能力放进一张新卡片里。

## Confirmed Scope

本期做：

- 保留 `文稿概览` 作为目录导航主卡片
- 保留显式关系卡片，并统一命名为 `上下文`
- 新增 `关联` 卡片
- 给目录中的彩色点定义固定语义
- `关联` 卡片支持文档态和块态
- 每条关联结果提供最小动作：`打开`、`建立链接`

本期不做：

- 顶部一级主视图切换
- 全局图谱首页
- 常驻迷你图
- 复杂多跳图筛选
- 自动主题命名
- 把搜索结果页重做成新工作台

## Recommended Sidebar Structure

右侧栏最终结构建议收敛为 3 张卡片：

1. `文稿概览`
2. `上下文`
3. `关联`

顺序保持从“结构”到“已知关系”再到“新发现关系”：

- `文稿概览`：看结构、跳位置
- `上下文`：看已经存在的关系
- `关联`：看系统推断和推荐出的关系

这是最符合当前信息心智的排列方式。

## Card Responsibilities

### 1. 文稿概览

`文稿概览` 的本质仍然是目录，不应被重做成第二个上下文面板。

它应该保持：

- 标题层级导航
- 当前阅读位置高亮
- 点击后跳转到对应标题或块

它可以适度增强，但增强必须从属于目录语义：

- 用彩色点表示少量状态
- 支持 `只看关键块` 这种轻量过滤
- 对当前焦点块进行更明确高亮

不建议在这里直接塞入标签、引用列表或推荐结果。

### 2. 上下文

`上下文` 专门承接显式、已存在的关系。

建议内容：

- 标签
- 提及文档 / 被提及于
- 所在路径
- 后续可扩展为文件夹位置、空间归属等稳定属性

用户看到这张卡片时，应该能明确理解：

“这里展示的是已经建立好的关系，不是系统猜出来的。”

### 3. 关联

`关联` 是本次新增卡片，承接 Graphify 型能力。

建议内容：

- 相关文档
- 相似段落
- 相关标签
- 推荐链接
- `查看关系图` 入口

用户看到这张卡片时，应该自然理解：

“这里展示的是系统发现的新关系或可采纳建议。”

## Outline Dot Semantics

目录右侧的彩色点建议保留，但必须从“漂亮装饰”升级成“有明确语义的状态标记”。

推荐语义如下：

- `绿色`：显式关系
  - 当前标题下存在引用 / 被引用 / 已有标签
- `紫色`：智能关联
  - 当前标题下存在相似段落、相关文档、推荐关系
- `蓝色`：当前焦点
  - 当前阅读块、搜索命中块或被强调的关键块

使用规则：

- 普通目录项不显示点
- 有价值状态的目录项才显示点
- 单个目录项最多显示一个主状态点
- 如果同一项同时满足多种状态，按优先级取一个：
  - 当前焦点
  - 智能关联
  - 显式关系

这能避免目录变成彩点墙，同时保住目录的导航感。

## Outline Visual Adjustments

目录样式建议做小幅调整，而不是重做。

建议：

- H1 / H2 / H3 的字号、缩进和字重差异更清楚
- 当前项高亮由“整行重色块”改成更克制的浅底色 + 细侧边强调
- 状态点尺寸保持很小，只作为信息提示，不抢正文
- `只看关键块` 做成轻量筛选，不做成强动作按钮

一句话原则：

先让它看起来像目录，再让它悄悄变聪明。

## Association Card Behavior

`关联` 卡片需要支持两种状态：

### 文档态

当用户只是打开文档，没有聚焦到某个具体块时，显示文档级关联：

- 相关文档
- 相关标签
- 主题邻居
- `查看关系图`

### 块态

当用户点击某个块、目录项，或在正文中形成明确焦点时，切换到块级关联：

- 相似段落
- 推荐建立链接
- 当前块对应的相关文档
- 推荐标签

状态切换原则：

- 用户无明确块焦点时，默认显示文档态
- 用户点目录项或正文块后，自动进入块态
- 焦点解除后，回到文档态

## Association Card Content Model

每条结果建议使用轻量卡片呈现，而不是纯文本列表。

文档卡片建议字段：

- 标题
- 所在文件夹或路径摘要
- 关系类型
- 相关度分数

段落卡片建议字段：

- 1 到 2 行摘要
- 来源文档标题
- 相似度分数
- 是否已建立显式链接

标签卡片建议字段：

- 标签名
- 关联文档数
- 是否已属于当前文档

每条记录的动作控制在最小集合：

- `打开`
- `建立链接`
- `忽略`（仅语义推荐项）

## Interaction Rules

关键联动建议如下：

- 点击 `文稿概览` 某一项：
  - 正文跳转到对应位置
  - `关联` 切换为该块的块态结果

- 光标或当前焦点块变化：
  - `关联` 自动刷新
  - 对应目录项状态高亮更新

- 点击 `上下文` 中显式关系项：
  - 打开目标文档或目标块

- 点击 `关联` 中推荐项：
  - 先打开对应目标
  - 若用户选择 `建立链接`，再把推荐关系升级为显式关系

这会形成一个清晰心智：

- 目录负责定位
- 上下文负责确认已有关系
- 关联负责探索新关系

## Search And Graph Entry

本期不把搜索页重做成主视图，但要为后续扩展留入口。

建议先只在 `关联` 卡片中预留一个按钮：

- `查看关系图`

首版可以打开：

- 一个轻量弹层
- 或一个局部关系页入口

它只围绕当前文档或当前块展示 1 跳子图，不直接承担完整全局图谱职责。

这样能在不重构导航的前提下，为后续图谱体验留出生长点。

## Data And System Implications

这个切片不要求先做完整图数据库产品，而是先让 UI 有稳定承载位。

对数据层的最小要求是：

- 继续使用现有文档、标签、显式引用数据
- 新增一个“关联结果”读取层，为 `关联` 卡片提供统一结果

建议返回结构至少区分：

- `documents`
- `blocks`
- `tags`
- `suggestedLinks`

这样前端卡片可以先稳定落位，后面再逐步把 Graphify/语义分析引擎接进去。

## MVP Sequence

### Step 1. Sidebar terminology and structure cleanup

- 把现有显式关系区统一命名成 `上下文`
- 保留 `文稿概览`
- 预留 `关联` 卡片位置

### Step 2. Outline state semantics

- 固定彩点语义
- 调整目录高亮与层级样式

### Step 3. Association card MVP

- 文档态先支持 `相关文档`、`相关标签`
- 块态先支持 `相似段落`
- 动作先支持 `打开`、`建立链接`

### Step 4. Graph entry

- `查看关系图` 先做局部子图入口

## Validation

最低验收标准：

- 右侧栏结构仍然清晰，目录价值没有被新能力挤掉
- 用户能一眼分辨 `上下文` 和 `关联` 的区别
- 目录彩点有固定语义，不再只是装饰
- 点击目录项后，`关联` 能刷新到对应块
- 不需要新增一级导航，也能完成 Graphify 型能力的首版接入

## Final Recommendation

这一期不要把 WorkKnowlage 直接推成一个“图谱主导界面”。

先做一个更稳的版本：

- 保住目录
- 保住显式关系
- 新增关联卡片
- 把目录彩点变成真正有语义的状态系统

如果这版体验成立，再继续往搜索关联、局部关系图和完整图谱页扩展，会自然很多。
