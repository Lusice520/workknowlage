## 3. 流程说明与流程图

本地分享的核心流程是“用户把当前文档临时开放给同一局域网内的其他设备只读访问”。流程需要同时保证两件事：生成的链接对其他设备可访问，而不是仅本机可访问；关闭分享后，旧链接应失效。刷新地址和关闭分享是该功能的关键分支流程，不能只作为按钮行为附带描述。

### 3.1 主流程：开启并访问分享

用户打开一个可读取的文档后，通过分享控件开启本地分享。系统启动 share server，并确保服务监听非 loopback 地址；随后系统选择当前机器可用的 LAN IP，生成分享链接并展示给用户。用户复制链接后，同一局域网内的其他设备可以通过浏览器访问只读分享页。该流程的成功结果是：分享页可读、不可编辑，正文、目录和常见内容块正常展示。

```mermaid
flowchart LR
    Doc["用户打开可读取文档"] --> Start["点击分享控件"]
    Start --> Server["系统启动 share server"]
    Server --> Listen["监听非 loopback 地址"]
    Listen --> Lan["获取可用 LAN IP"]
    Lan --> Url["生成分享链接"]
    Url --> Copy["用户复制链接"]
    Copy --> Visit["同一局域网设备访问"]
    Visit --> Readonly["展示只读分享页"]
```

### 3.2 分支流程：刷新分享地址

当用户网络环境变化、当前 IP 不可用，或用户主动希望重新确认分享地址时，可以触发刷新地址。系统重新读取本机可用 LAN IP，并更新当前分享 URL。刷新不应重新创建文档内容，也不应破坏已开启的分享状态；如果没有可用 LAN IP，系统应保留原状态并给出明确提示。

```mermaid
flowchart LR
    Refresh["用户点击刷新地址"] --> Detect["系统重新检测 LAN IP"]
    Detect --> HasIp{"找到可用地址?"}
    HasIp -->|是| Update["更新分享 URL"]
    Update --> Show["展示新地址并允许复制"]
    HasIp -->|否| Keep["保留当前分享状态"]
    Keep --> Warn["提示未找到可用局域网地址"]
```

### 3.3 分支流程：关闭分享与失效访问

用户完成临时交付后，可以关闭当前文档分享。系统停止或撤销当前分享状态，使旧链接不可继续访问。关闭后，如果其他设备再次访问旧链接，应看到失效或不可访问提示，而不是继续看到旧内容。该流程的关键要求是：关闭动作必须真实影响访问结果，不能只是 UI 状态变化。

```mermaid
flowchart LR
    Close["用户关闭分享"] --> Stop["系统停止当前分享"]
    Stop --> Invalidate["旧链接失效"]
    Invalidate --> Revisit["其他设备访问旧链接"]
    Revisit --> Expired["显示分享已关闭或不可访问"]
```

## 4. 特殊业务

1. 分享链接不能只显示 `127.0.0.1`，否则其他设备无法访问。
2. share server 不能只监听 loopback 地址，必须监听非 loopback 地址，例如 `0.0.0.0`。
3. 分享范围是同一局域网，不承诺公网访问。
4. 分享页必须只读，不提供编辑入口。
5. 关闭分享后，旧链接不可继续访问。
6. 分享页宽屏布局要让正文和目录形成 centered reading group，不能让目录像孤岛一样挂在右侧。
7. 不能只放大白色内容卡片来假修复布局；需要同时检查正文宽度、目录宽度、grid gap 和整体居中。
8. 长中文标题、长目录、富表格、提醒块和列表都应保持可读。

## 5. 页面 / 状态说明

| 页面 / 状态 | 说明 | 可用操作 |
|---|---|---|
| 未开启 | 当前文档没有分享链接 | 开启分享 |
| 启动中 | 用户已触发分享，系统正在启动服务和生成链接 | 禁用重复开启，可取消或等待 |
| 已开启 | 当前文档已生成 LAN IP 分享链接 | 复制链接、刷新地址、关闭分享 |
| 启动失败 | share server 启动失败或地址不可用 | 重试、查看错误 |
| 已关闭 | 用户主动关闭分享 | 重新开启分享 |
| 分享页只读访问 | 浏览器展示文档内容和目录 | 阅读、滚动、点击目录 |
| 分享失效访问 | 分享关闭后访问旧链接 | 显示失效提示 |

## 9. 交互说明

| 交互 | 说明 |
|---|---|
| 页面加载 | 如果当前文档已有分享状态，展示当前状态和分享地址 |
| 开启分享 | 启动服务并生成 LAN IP 链接；成功后进入“已开启”状态 |
| 复制链接 | 将当前完整分享 URL 写入剪贴板，并给出成功提示 |
| 刷新地址 | 重新获取本机 LAN IP；如果 IP 变化，更新展示链接 |
| 关闭分享 | 停止当前分享；成功后进入“已关闭”或“未开启”状态 |
| 访问分享页 | 浏览器展示只读文档、目录、正文和基础样式 |
| 分享失效 | 分享关闭后访问旧 URL，展示失效或不可访问提示 |

## 10. 提示说明

| 场景 | 提示类型 | 提示文本 |
|---|---|---|
| 开启成功 | 全局提示 / 控件状态 | 分享已开启 |
| 复制成功 | 全局提示 | 分享链接已复制 |
| 刷新成功 | 控件状态 | 分享地址已刷新 |
| 关闭成功 | 全局提示 / 控件状态 | 分享已关闭 |
| 端口占用 | 错误提示 | 分享服务启动失败，端口可能被占用 |
| 无 LAN IP | 错误提示 | 未找到可用于局域网访问的地址，请检查网络连接 |
| 分享失效 | 分享页提示 | 当前分享已关闭或不可访问 |

## 11. 异常处理

| 异常场景 | 系统处理 | 用户反馈 | 是否阻塞 |
|---|---|---|---|
| share server 启动失败 | 停止启动流程，保留文档状态 | 显示启动失败原因 | 是 |
| 端口被占用 | 尝试可用端口或提示失败 | 显示端口占用相关提示 | 是 |
| 无可用 LAN IP | 不生成误导性的 `127.0.0.1` 对外链接 | 提示检查网络或刷新地址 | 是 |
| 文档内容渲染失败 | 尽量保留纯文本可读内容 | 分享页显示部分内容和错误说明 | 部分阻塞 |
| 分享关闭后访问 | 返回失效状态 | 显示分享已关闭 | 是 |
| 分享页布局异常 | 不影响服务，但需要记录为视觉缺陷 | 无用户错误提示，研发侧修复 | 否 |

## 13. 权限与边界

1. 本地分享不提供公网访问能力。
2. 本地分享不提供编辑权限。
3. 当前版本不提供访问密码、有效期、访问日志和权限控制。
4. 当前版本面向同一局域网临时阅读，不等同于发布或协作。

## 14. 验收标准

1. 开启分享后，分享链接使用 LAN IP，而不是仅使用 `127.0.0.1`。
2. share server 监听非 loopback 地址，同一局域网内其他设备可以访问。
3. 关闭分享后，旧链接不可继续访问。
4. 分享页能展示标题、时间、正文、提醒块、列表、表格和目录。
5. 长中文标题不会挤压布局或溢出。
6. 宽屏下正文和目录作为整体阅读组居中，不出现明显视觉偏斜。
7. 分享控件不拆成多个冗余按钮，状态和主操作清晰可读。
8. 启动失败、端口占用、无 LAN IP 时有明确提示。
9. 相关渲染或分享服务改动有 focused tests 或 browser smoke 验证。
