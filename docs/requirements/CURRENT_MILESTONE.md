> 最后更新：2026-05-18

> 当前阶段：M1 本地长用稳定版收尾，M2 结构化知识增强已部分启动  
> 关联 PRD：`docs/requirements/PRD.md`

## 1. 当前判断

WorkKnowlage 已经具备本地知识工作台的主要骨架：文档树、编辑器、本地持久化、搜索、导出、局域网分享、右侧栏关系导航和 macOS DMG 打包都已有可运行基础。当前阶段不应继续只围绕单点 UI 或单点算法发散，而应把工作分成两条线：

1. **M1 收尾**：补齐 P0 高风险能力的 SPEC、验证和发布记录，让本地长用稳定版更可交付。
2. **M2 启动**：围绕结构化知识增强推进模板、元数据、Wiki 关联质量和未来图谱候选。

右侧 Wiki 关联已从“相似内容发现”推进到“推荐解释 + 全部线索 + 轻反馈 + 智能判定边界”。这说明 M2 的关系导航方向已经开始，但仍不应把大模型能力混入 P0 主流程。

## 6. 推荐下一步

建议下一阶段优先级：

1. **补齐发布与本地数据 SPEC**：`macos_dmg_spec.md`、`local_data_spec.md`。这两类风险最高，直接影响“可交付”和“内容可靠”。
2. **整理 0.4.2 之后的版本计划**：确认下一包是否为 0.4.3，并更新 release notes 模板。
3. **为 M2 建第一个结构化增强 SPEC**：优先从文档模板或元数据二选一，不要同时展开太多。
4. **保留 Wiki 关联智能判定为边界能力**：当前先不接真实模型；等模型配置、数据发送边界和用户授权明确后再进入实现。

## 7. 阶段验收口径

M1 可以认为进入“可长期自用”状态的条件：

1. P0 主流程可连续使用：创建、编辑、保存、搜索、关联、导出、分享、重开。
2. 高风险功能均有 SPEC 或明确验证记录。
3. 全量测试、typecheck、build、package 可通过。
4. DMG 可安装启动。
5. release notes 记录版本、验证命令、产物路径和已知限制。

M2 启动的条件：

1. 不影响 M1 稳定性。
2. 每个结构化增强功能先有 SPEC。
3. 智能能力不绕过数据边界、用户授权和可解释证据。
