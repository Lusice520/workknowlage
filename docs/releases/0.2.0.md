# WorkKnowlage 0.2.0

**Release date:** 2026-05-06
**Stage:** P1-1 Boundary Hardened

This is the first recommended long-use local build after the P1-1 roadmap work. The release focuses on making the app/session/runtime API seams stable enough for daily use and future iteration.

## Highlights

- App/session/shell boundaries are now protected by a dedicated P1-1 contract test.
- Runtime API access is kept out of feature UI modules and routed through app hooks or shared adapters.
- Browser mock/runtime implementation details are split out of the thin `workKnowlageApi` entrypoint.
- Quick note record loading is centralized in `shared/lib/quickNoteRecords`.
- Duplicate tag chips were removed from the center pane metadata row.
- Rich-table deferred callbacks now clean up after editor/test teardown.
- The local macOS app can be packaged from this revision with `npm run package:mac` or `npm run package:mac:dir`.

## Verification

- `npm test`
- `npm run build`

## Known Notes

- The macOS local build uses ad-hoc signing unless developer signing/notarization credentials are configured.
- Vite still reports large editor/export chunks. This is a performance optimization follow-up, not a release blocker.
- P1-3 sidebar association data-layer decoupling is documented separately and intentionally not completed in this release.
